k8s-sentinel

A lightweight Kubernetes CronJob that heals your self-hosted cluster before you wake up — disk pressure, containerd CRI, kube-proxy, and more — with optional Ansible SSH fixes.

Safe default: autoFix off

GitHub v0.2.7 Release Install guide

The problem

3 AM disk full

DiskPressure evicts workloads. Prometheus pages you; SSH is still manual.

containerd CRI drift

Registry mirror config breaks the CRI plugin — nodes go Unknown.

System pods flapping

kube-proxy and CoreDNS need restarts; cloud agents rarely SSH to metal.

How it works

CronJob (30m) → 7 check modules → metrics + JSON → optional kubectl fix → optional Ansible SSH

Quick start (check-only)

helm upgrade --install k8s-sentinel oci://ghcr.io/dejavux/charts/k8s-sentinel \
  --version 0.2.7 \
  -n kube-system

kubectl create job --from=cronjob/k8s-sentinel sentinel-check-$(date +%s) -n kube-system
